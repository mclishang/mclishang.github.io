这是一篇用于发布验证的测试文章。它会尽量覆盖博客中常见的内容元素，方便检查首页摘要、文章详情页、目录、代码高亮、数学公式、引用和表格样式是否符合预期。

## 文章目标

这篇文章主要用于完成三件事：

1. 验证文章列表是否能正常展示标题、摘要、分类和发布日期。
2. 验证正文排版是否清晰，包括段落、强调、列表、引用和代码块。
3. 验证扩展内容是否正常渲染，例如表格、公式和分节目录。

如果这一篇文章显示正常，后续继续发布正式内容时就会更稳妥。

## 列表与结构化内容

在写博客时，列表经常用于表达步骤、要点或者对比结论。一个好的列表应该简洁、并列且易于扫描。

### 无序列表

- 内容结构尽量清晰。
- 小节标题尽量语义明确。
- 示例内容要足够真实，不能只有空壳。

### 有序列表

1. 先确定主题。
2. 再整理结构。
3. 最后补充示例与总结。

## 代码示例

下面放一段简单的 TypeScript 代码，用来检查代码块的字体、缩进、滚动和高亮效果：

```ts
type ArticleMeta = {
  title: string
  slugId: string
  draft?: boolean
}

export function formatArticle(meta: ArticleMeta) {
  return `${meta.title} (${meta.slugId})`
}
```

如果代码块在移动端也能正常横向滚动，不会挤压页面布局，那么阅读体验会更稳定。
